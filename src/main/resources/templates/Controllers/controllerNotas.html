Notas
TYPE CONTROLLER MAPPER ENDPOINT VARIABLES Descripcion EJEMPLO
GET /nota /usuario / {username} Nos trae una lista segun el {username} del sistema ip:port/nota/usuario/elMonedita
GET /nota /titulo / {titulo} Nos trae una lista de notas si en el titulo contiene al menos el {titulo} ingresado ip:port/nota/titulo/tangacorta
GET /nota /texto / {texto} Nos trae una lista de notas si en el texto contiene al menos el {texto} ingresado. NOTA 1 ip:port/nota/texto/rompio%20el%20vidrio
GET /nota /fecha / {fecha} Devuelve todas las notas en que correspondan a la fecha yyyy-MM-dd ip:port/nota/fecha/2020-10-21
GET /nota /fecha/hora/ / {querys} Va a devolver todas las notas entre dos timestamp enviados. AUN NO IMPLEMENTADO ip:port/nota/fecha/hora/
POST /nota / NONE NONE Ingresa una nota nueva asociadaa al usuario enviado, chequeaando el mismo esta activo (consultar esto)

Ejemplo:

{ "nota": { "titulo":"Nota test" , "texto":"postaasdasdsad" }, "user":{ "login":"jch321", "password":"1234" } 		ip:port/nota/
PUT /nota / NONE NONE Modifica una nota existente, modifica la nota con ID: 1

Ejemplo:

{ "id": 1, "titulo": "Nota test", "texto": "texto modificado3", } 		ip:port/nota/
